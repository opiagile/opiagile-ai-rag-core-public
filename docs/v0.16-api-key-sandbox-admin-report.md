# Relatório v0.16 — Liberação Controlada De API Key Sandbox

Data da sprint: 2026-06-30

## Resumo Executivo

A v0.16 adiciona um fluxo administrativo mínimo para transformar solicitações do portal `/developers` em API keys sandbox controladas.

Antes, o formulário registrava a solicitação no banco, mas a criação da chave dependia de operação manual direta. Agora existe um endpoint administrativo que:

- lista solicitações developer;
- aprova uma solicitação;
- cria um registro em `api_clients`;
- retorna a API key uma única vez;
- grava somente prefixo e hash da chave;
- marca a solicitação como `APPROVED`.

## Segurança

- Endpoints administrativos exigem `X-Demo-Admin-Token`.
- A API key gerada não é salva em texto claro.
- A chave completa aparece apenas na resposta de aprovação.
- Reemissão é bloqueada quando a solicitação já está `APPROVED`.
- Endpoints administrativos foram marcados com `@Hidden` para não aparecerem no OpenAPI público do portal developers.
- O uso recomendado é via acesso operacional restrito, SSH/túnel ou chamada interna, não como fluxo público aberto.

## Endpoints Adicionados

```text
GET  /api/admin/developer-access-requests
POST /api/admin/developer-access-requests/{requestId}/approve
```

## Exemplo De Aprovação

```bash
curl -X POST http://localhost:8080/api/admin/developer-access-requests/<request-id>/approve \
  -H "Content-Type: application/json" \
  -H "X-Demo-Admin-Token: <admin-token>" \
  -d '{
    "tenantSlug": "demo",
    "workspaceSlug": "clinica-demo",
    "scopes": ["chat:write", "documents:upload", "documents:read", "providers:read", "workspaces:read"],
    "rateLimitPerMinute": 30,
    "clientName": "Sandbox - Empresa Demo"
  }'
```

## Limitações
