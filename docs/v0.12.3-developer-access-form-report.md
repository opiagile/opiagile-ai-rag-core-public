# Relatório v0.12.3 — Formulário De Solicitação De API Key

## Resumo Executivo

A v0.12.3 substitui o `mailto` do portal `/developers` por um diálogo com formulário real para solicitar API key sandbox.

O formulário registra a solicitação no banco do core. A geração da API key continua manual, para controlar tenant, workspace, escopos e limite por minuto antes de liberar acesso.

## O Que Foi Implementado

- Diálogo nativo no portal `/developers`.
- Formulário com:
  - nome;
  - empresa;
  - email;
  - objetivo do teste;
  - recursos pretendidos.
- Endpoint público:

```text
POST /api/developer-access-requests
```

- Persistência em:

```text
developer_access_requests
```

- Rate limit simples por IP.
- Campo invisível antispam.
- Validação de email e tamanho dos campos.
- Endpoint oculto do OpenAPI comercial para manter o console focado no contrato RAG.

## Segurança E Operação

- Nenhuma API key é gerada automaticamente.
- Nenhum segredo é exibido ou salvo pelo formulário.
- O usuário recebe confirmação de registro, não uma chave.
- O core ainda não envia email automático para novas solicitações.
- Para receber notificação automática, o próximo incremento recomendado é integrar Microsoft Graph, SMTP ou uma fila operacional de leads.

## Consulta Operacional

Enquanto não houver painel administrativo ou notificação por email, as solicitações podem ser consultadas no banco:

```sql
SELECT created_at, status, name, company, email, use_case, requested_resources
FROM developer_access_requests
ORDER BY created_at DESC
LIMIT 50;
```
