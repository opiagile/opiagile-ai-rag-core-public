# Caso De Portfólio

## Problema

Empresas respondem perguntas repetitivas todos os dias, mas o conhecimento costuma estar espalhado em PDFs, FAQs, mensagens antigas e regras internas.

## Solução

Este projeto demonstra um assistente de atendimento com IA/RAG que consulta documentos, responde com fontes, mantém histórico, qualifica leads e encaminha para humano quando necessário.

## Diferenciais Demonstrados

- Backend Java/Spring Boot estruturado por domínio.
- RAG demonstrável com fontes, PostgreSQL/pgvector preparado para embeddings reais e modo local sem chave externa.
- Modo demonstração sem chave externa.
- Triagem de intenção e lead status.
- Webhook estilo WhatsApp com payload mock.
- Workflows n8n.
- Observabilidade local por conversa.

## Como Usar Em Proposta

"Tenho um core público demonstrando RAG com IA: upload de documentos, embeddings opcionais, recuperação pgvector, respostas com fontes, memória conversacional, triagem, handoff humano, observabilidade e contrato para frontend/gateways separados. Posso adaptar essa base ao seu fluxo e documentos."
