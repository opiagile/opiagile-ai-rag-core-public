# Relatório v0.12.2 — Console API Com Navegação Lateral

## Resumo Executivo

A v0.12.2 substitui a experiência principal de console em `/developers/console` por uma interface própria, leve e orientada a uso comercial/técnico.

O Swagger UI customizado foi preservado em `/developers/swagger-ui/index.html` como fallback técnico, mas o fluxo recomendado agora é o console Opiagile com menu lateral, painel central de parâmetros e execução direta dos endpoints.

## O Que Foi Implementado

- Novo console estático em:

```text
GET /developers/api-console/index.html
GET /developers/console
```

- Menu lateral com:
  - seleção de servidor;
  - campo de API key;
  - consulta de workspaces permitidos pela chave;
  - busca por endpoint;
  - agrupamento por tag do OpenAPI.
- Painel central com:
  - método e path;
  - parâmetros de path, query e header;
  - corpo JSON com exemplo gerado a partir do schema;
  - suporte básico a `multipart/form-data`;
  - execução por `fetch`;
  - preview de cURL com API key mascarada;
  - resposta formatada com status HTTP e tempo.
- Fallback preservado:

```text
GET /developers/swagger-ui/index.html
```

## Decisões Técnicas

- O console lê `GET /v3/api-docs/rag-core` em tempo de execução.
- Não foi adicionada dependência externa de UI.
- A API key fica somente em `sessionStorage` do navegador.
- O contrato OpenAPI público lista apenas `https://opiagile.com`; exemplos locais continuam documentados separadamente.
- Servidores `localhost` são ocultados no seletor se aparecerem em ambiente de desenvolvimento.
- A consulta de workspaces usa `GET /api/workspaces` com `X-OPIAGILE-API-KEY`.
- O preview de cURL mascara `X-OPIAGILE-API-KEY`.
- O core continua sem frontend de produto; este console é documentação interativa para desenvolvedores.
- O portal `/developers` informa que a API key sandbox é liberada manualmente por email, com tenant, workspace, escopos e limite por minuto definidos pela Opiagile.
- O item "Como testar" foi ajustado para orientar o console Opiagile, que usa campo próprio de API key. O botão `Authorize` permanece apenas no Swagger clássico.

## Segurança

- Nenhum segredo real foi incluído.
- A chave não é salva em backend, cookie ou arquivo.
- A chave não aparece completa no cURL renderizado.
- A recomendação continua sendo usar API keys server-to-server, com chaves sandbox para testes no console.
