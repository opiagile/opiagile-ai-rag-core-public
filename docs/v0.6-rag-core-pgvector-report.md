# Relatório v0.6 — RAG Core Com Embeddings E pgvector

## Resumo Executivo

A v0.6 reposiciona o projeto como `opiagile-ai-rag-core` e implementa o fluxo de embeddings reais com recuperação pgvector, mantendo fallback textual local sem chave externa.

O piloto WhatsApp Cloud API entregue nas versões anteriores foi preservado como referência técnica, mas deixa de ser a linha principal deste repositório. A evolução recomendada de canais externos passa a ser por aplicações gateway separadas, começando por `opiagile-whatsapp-ai-gateway`.

## Decisões

- Este repositório fica focado em RAG, fontes, memória, LLM opcional, observabilidade, avaliação e contrato HTTP.
- WhatsApp, Teams, Slack, CRM e outros canais devem consumir o core por HTTP.
- Código existente de WhatsApp não foi removido para preservar histórico, testes e referência de extração.
- O modo local continua funcionando sem `OPENAI_API_KEY`.
- Embeddings reais só são usados quando `OPENAI_EMBEDDINGS_ENABLED=true` e `OPENAI_API_KEY` está configurada.

## O Que Foi Implementado

- `OpenAiEmbeddingProvider` real, usando endpoint de embeddings da OpenAI.
- Configuração `OPENAI_EMBEDDING_DIMENSIONS`, padrão `1536`.
- Persistência de embeddings em `document_chunks.embedding`.
- Recuperação pgvector por similaridade de cosseno quando a consulta possui embedding.
- Fallback textual local quando:
  - embeddings estão desabilitados;
  - a chave OpenAI não está presente;
  - chunks não possuem vetor;
  - a consulta vetorial não retorna fontes.
- Registro do provider de recuperação nos chunks retornados para trace/log.
- Testes unitários para formato pgvector e roteamento de recuperação.
- Contrato para gateways em `docs/gateway-contract.md`.

## Como Ativar Embeddings Reais

Configurar no `.env` local ou ambiente seguro:

```text
OPENAI_EMBEDDINGS_ENABLED=true
OPENAI_API_KEY=<valor seguro fora do Git>
OPENAI_EMBEDDING_MODEL=text-embedding-3-small
OPENAI_EMBEDDING_DIMENSIONS=1536
```

Depois, reenviar os documentos para que os chunks sejam gravados com vetor. Chunks antigos sem embedding continuarão disponíveis para fallback textual, mas não participarão da consulta pgvector.

## Limitações Mantidas

- O projeto ainda não tem avaliação RAG automatizada.
- O modo pgvector depende de documentos ingeridos após habilitar embeddings.
- O piloto WhatsApp preservado não deve ser tratado como produção.
- Demo pública com HTTPS, autenticação e limites ainda é pendente.
- Canais reais devem evoluir fora deste core.
