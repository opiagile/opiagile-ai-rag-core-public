# Relatório v0.16.1 — Painel Interno De Administração

Data da sprint: 2026-06-30

## Resumo Executivo

A v0.16.1 adiciona um painel interno, acessível apenas via túnel SSH, para operar o fluxo de aprovação de API keys sandbox criado na v0.16.

O painel não foi exposto em domínio público. Ele é servido pelo Caddy em `127.0.0.1:9090` na VPS Oracle e usa proxy interno para `/api/admin/*`.

## O Que Foi Implementado

- Painel estático em `deploy/oracle-free-tier/admin-panel/`.
- Porta `9090` publicada somente em `127.0.0.1`.
- Bloco Caddy interno para servir o painel e rotear `/api/admin/*` para a API.
- Bloqueio explícito de `/api/admin/*` no catch-all HTTP usado por acesso direto ao IP.
- Documentação de acesso via túnel SSH.

## Segurança

- O painel não fica disponível em `opiagile.com`.
- A porta `9090` fica bindada apenas no loopback da VPS.
- O acesso externo depende de túnel SSH autorizado.
- O token `X-Demo-Admin-Token` continua obrigatório.
- Nenhum segredo foi embutido no painel.
- A API key sandbox é exibida somente uma vez após aprovação.
- O acesso direto por IP em HTTP responde `404` para `/api/admin/*`.

## Como Acessar

```bash
ssh -i ~/.ssh/sua_chave_oracle -L 9090:localhost:9090 ubuntu@<IP_PUBLICO_DA_VPS>
```

Depois:

```text
http://localhost:9090
```
