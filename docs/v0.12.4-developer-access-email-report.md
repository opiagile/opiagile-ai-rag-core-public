# Relatório v0.12.4 — Email Para Solicitações De API Key

> Atualização: a v0.17 evoluiu este fluxo para outbox transacional, confirmação ao lead, notificação interna e entrega da API key por link de uso único. Consulte `docs/v0.17-email-entrega-api-key-report.md` para o estado atual.

## Resumo Executivo

A v0.12.4 adiciona notificação opcional e assíncrona por email para solicitações de API key sandbox feitas no portal `/developers`.

Cada solicitação é salva primeiro em `developer_access_requests`. Um scheduler tenta enviar o email depois, marca sucesso com `notification_email_sent=true` e mantém falhas pendentes para retry. A geração da API key continua manual.

Atualização operacional: o scheduler envia notificação apenas para solicitações ainda em `NEW`. Solicitações já `APPROVED`, `REJECTED`, `ARCHIVED` ou `CONTACTED` não geram aviso atrasado, mesmo que tenham `notification_email_sent=false`.

## O Que Foi Implementado

- Dependência `spring-boot-starter-mail`.
- Configuração SMTP via variáveis de ambiente.
- Serviço `DeveloperAccessNotificationService`.
- Propriedades `DeveloperAccessEmailProperties`.
- Envio assíncrono por `DeveloperAccessEmailScheduler`.
- Campos persistidos para fila de email, tentativas, próximo retry, último erro e data de envio.
- Testes unitários para envio, modo desabilitado e ausência de `JavaMailSender`.

## Comportamento

- Se `DEVELOPER_ACCESS_EMAIL_ENABLED=false`, o scheduler não processa a fila e não consome tentativas.
- Se SMTP não estiver configurado, a solicitação continua salva no banco.
- Se o envio falhar, o formulário continua respondendo com sucesso e a solicitação fica com `notification_email_sent=false` para nova tentativa.
- Registros já existentes antes da migration V15 são marcados como tratados para evitar disparos retroativos.
- O health check SMTP fica desabilitado por padrão para não deixar `/actuator/health` indisponível quando o email ainda não estiver configurado.
- Nenhum token, senha ou segredo é impresso em logs.

## Variáveis De Ambiente

```text
DEVELOPER_ACCESS_EMAIL_ENABLED=true
DEVELOPER_ACCESS_EMAIL_TO=contato@opiagile.com
DEVELOPER_ACCESS_EMAIL_FROM=
DEVELOPER_ACCESS_EMAIL_SUBJECT_PREFIX="[Opiagile] Solicitação de API key sandbox"
SPRING_MAIL_HOST=smtp.office365.com
SPRING_MAIL_PORT=587
SPRING_MAIL_USERNAME=
SPRING_MAIL_PASSWORD=
SPRING_MAIL_SMTP_AUTH=true
SPRING_MAIL_SMTP_STARTTLS_ENABLE=true
SPRING_MAIL_HEALTH_ENABLED=false
DEVELOPER_ACCESS_EMAIL_SCHEDULER_ENABLED=true
DEVELOPER_ACCESS_EMAIL_SCHEDULER_FIXED_DELAY_MS=60000
DEVELOPER_ACCESS_EMAIL_SCHEDULER_BATCH_SIZE=5
DEVELOPER_ACCESS_EMAIL_SCHEDULER_MAX_ATTEMPTS=10
DEVELOPER_ACCESS_EMAIL_SCHEDULER_RETRY_DELAY_MINUTES=15
```

Para Microsoft 365, `SPRING_MAIL_USERNAME` e `SPRING_MAIL_PASSWORD` devem ser definidos fora do Git. Se `contato@opiagile.com` for alias ou shared mailbox, valide se a conta usada tem permissão para enviar como esse endereço.
