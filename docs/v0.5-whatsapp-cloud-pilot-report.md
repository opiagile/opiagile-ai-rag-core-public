# Relatório v0.5 — WhatsApp Cloud API Piloto Controlado

## Resumo Executivo

A v0.5 adiciona suporte ao piloto controlado da WhatsApp Cloud API oficial da Meta, mantendo o webhook mock local e preservando o modo de demonstração sem credenciais.

O projeto agora suporta três níveis de operação: `MOCK`, `META_CLOUD` em dry-run e `META_CLOUD` com envio real controlado. O envio real permanece bloqueado por padrão e depende de credenciais externas, assinatura válida, HTTPS público, allowlist e consentimento dos testers.

## Estado Anterior

Antes desta sprint, o projeto tinha webhook estilo WhatsApp mock em `POST /api/webhooks/whatsapp`, mas não tratava o fluxo oficial da Meta Cloud API.

## O Que Foi Implementado

- Configuração tipada para WhatsApp.
- Endpoint de verificação da Meta: `GET /api/webhooks/whatsapp/meta`.
- Endpoint de recebimento Meta: `POST /api/webhooks/whatsapp/meta`.
- Validação de assinatura HMAC SHA-256 via `X-Hub-Signature-256`.
- Parser de payload Meta para mensagens de texto.
- Ignorar status updates e eventos não textuais.
- Allowlist de números de teste.
- Rate limit simples por telefone.
- Provider Meta Cloud API com dry-run padrão.
- Endpoint de status sem segredos: `GET /api/webhooks/whatsapp/status`.
- Persistência de eventos de webhook em `whatsapp_webhook_events`.
- Seção de status WhatsApp na Demo UI.
- Documentação do piloto e exemplos de API.

## Modos Suportados

- `MOCK`: local, sem credenciais reais.
- `META_CLOUD` dry-run: valida webhook/payload/allowlist/rate limit e chama o `ChatService`, mas não envia resposta real.
- `META_CLOUD` envio real: exige `WHATSAPP_SEND_ENABLED=true`, `WHATSAPP_DRY_RUN=false`, token, Phone Number ID, assinatura válida e número autorizado.

## Variáveis De Ambiente

```text
WHATSAPP_PROVIDER=MOCK
WHATSAPP_VERIFY_TOKEN=
WHATSAPP_APP_SECRET=
WHATSAPP_ACCESS_TOKEN=
WHATSAPP_PHONE_NUMBER_ID=
WHATSAPP_BUSINESS_ACCOUNT_ID=
WHATSAPP_GRAPH_API_VERSION=v23.0
WHATSAPP_ALLOWED_TEST_NUMBERS=
WHATSAPP_PUBLIC_BASE_URL=
WHATSAPP_SEND_ENABLED=false
WHATSAPP_DRY_RUN=true
WHATSAPP_SIGNATURE_REQUIRED=true
WHATSAPP_RATE_LIMIT_PER_MINUTE=5
WHATSAPP_BLOCK_UNAUTHORIZED=true
```

## Segurança Implementada

- Dry-run ativo por padrão.
- Envio real desabilitado por padrão.
- Assinatura obrigatória por padrão.
- Allowlist obrigatória para envio real.
- Rate limit por telefone.
- Logs e status sem tokens, app secret ou verify token.
- Telefone mascarado nos eventos operacionais.

## Limitações

- Precisa de HTTPS público para a Meta chamar o webhook.
- Envio real depende de credenciais Meta e configuração no painel da Meta.
- O piloto não é produção.
- Apenas mensagens de texto são processadas nesta sprint.
- Mídia, áudio, documentos, botões, templates e grupos ficam para depois.
- O envio real deve ser liberado inicialmente para 2 a 5 testers autorizados.

A implementação do piloto Cloud API está concluída, mas o envio real com tester ainda é uma pendência operacional. O primeiro teste real deve começar em dry-run, com HTTPS público controlado, assinatura válida, allowlist mínima e consentimento do responsável.
