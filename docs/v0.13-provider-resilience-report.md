# Relatório v0.13 — Resiliência De Provedores IA

## Resumo Executivo

A v0.13 adiciona visibilidade operacional segura para provedores de IA do core RAG. O objetivo é reduzir risco de dependência cega de um único provider, documentar fallback e preparar a próxima refatoração v0.14 com Spring AI.

Nenhum segredo é retornado. O endpoint não chama OpenAI, não gera custo e apenas mostra configuração efetiva e capacidade operacional.

## O Que Foi Implementado

- Novo endpoint:
  - `GET /api/providers/status`
- Novo pacote:
  - `backend/src/main/java/com/opiagile/supportai/provider`
- Status seguro para:
  - chat LLM;
  - embeddings;
  - estratégia de recuperação;
  - planner LLM de ferramentas controladas;
  - avisos de fallback.
- Documentação no README e exemplos de API.
- Inclusão do endpoint no grupo OpenAPI `rag-core`.
- Testes unitários para os principais cenários de provider/fallback.

## Modos Cobertos

### DEMO Sem Chave

- Chat usa `DEMO`.
- Embeddings ficam `NOOP`.
- Recuperação usa `LOCAL_TEXT`.
- Planner LLM fica inativo se não houver chave.

### OpenAI Solicitado Sem Chave

- Chat cai para fallback `DEMO`.
- Embeddings caem para fallback textual.
- Recuperação fica em `LOCAL_TEXT`.
- Avisos explicam a configuração incompleta.

### OpenAI Configurado

- Chat indica `OPENAI`.
- Embeddings indicam provider configurado.
- Recuperação indica `PGVECTOR_COM_FALLBACK_TEXTUAL`.
- Planner LLM indica `OPENAI` quando habilitado.

## Endpoint

```text
GET /api/providers/status
```

O endpoint retorna:

- `status`: `OK` ou `ATENCAO`;
- `chat`: provider solicitado, provider ativo, modelo, fallback e chave configurada como boolean;
- `embeddings`: provider ativo, modelo, dimensão, fallback e chave configurada como boolean;
- `retrieval`: estratégia ativa e se pgvector está pronto por configuração;
- `toolPlanner`: provider ativo do planner;
- `warnings`: avisos operacionais sem segredos.

## Segurança

O endpoint não retorna:

- `OPENAI_API_KEY`;
- tokens;
- passwords;
- payloads;
- prompts;
- documentos;
- telefones;
- dados pessoais.

Ele também não faz chamada externa. Portanto, pode ser usado em diagnóstico, portal de developers e suporte sem gerar custo.

## Limitações

- A v0.13 não adiciona provider local real.
- A v0.13 não migra para Spring AI.
- A v0.13 não altera fallback de runtime além do comportamento já existente.
- Rate limit continua em memória.
- pgvector continua com SQL manual por controle de tenant/workspace.
