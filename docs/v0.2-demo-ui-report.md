# Relatório Da Sprint Demo UI v0.2

Data: 2026-05-28
Projeto: `opiagile-ai-support-rag-starter`

## Resumo Executivo

A Sprint Demo UI v0.2 criou uma interface visual local para demonstrar upload de documento, chat com RAG, fontes, intenção, `conversationId`, handoff e trace básico.

A `landing/` foi preservada como página institucional. A nova `demo-ui/` é a aplicação funcional de demonstração local para cliente não técnico.

O backend não foi alterado nesta sprint. O projeto continua não sendo demo pública aberta, ambiente produtivo, integração real com WhatsApp ou RAG vetorial completo.

Frase de posicionamento mantida:

> RAG demonstrável com fontes, PostgreSQL/pgvector preparado para embeddings reais e modo local sem chave externa.

## Endpoints Consumidos

- `GET /actuator/health`
- `GET /api/version`
- `POST /api/documents/upload`
- `GET /api/documents`
- `GET /api/documents/{documentId}`
- `GET /api/documents/{documentId}/chunks`

- `GET /api/handoffs`
- `GET /api/observability/conversations/{conversationId}/trace`

O upload usa `documentId` como identificador principal retornado pela API.

## Decisões Técnicas

- Criada `demo-ui/` separada de `landing/` para manter a landing institucional limpa.
- Usado Vite com JavaScript vanilla para evitar dependências pesadas.
- Configurado proxy local para `/api` e `/actuator`, evitando alteração de CORS no backend.
- API base configurável via `VITE_API_BASE_URL` e `VITE_ACTUATOR_BASE_URL`.
- Nenhum schema, migration, entidade ou regra de negócio foi alterado.
- `vite@6.4.2` foi usado por ser compatível com Node 18 e por deixar `npm audit` sem vulnerabilidades conhecidas no momento da validação.
