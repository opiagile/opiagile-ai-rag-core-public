# v0.18.1 - Automação de Segurança

Data: 2026-07-01

## Resumo Executivo

A v0.18.1 transforma os próximos passos do pentest inicial em automações recorrentes.

Foram adicionados:

- Dependabot para Maven, Docker e GitHub Actions;
- workflow semanal/manual de análise de segurança;
- smoke test público não destrutivo para validar headers e bloqueios;
- validação de Caddy no workflow de `develop`;
- smoke público após deploy Oracle;
- testes unitários para IP confiável via `X-Real-IP`.

## Dependabot

O Dependabot foi configurado para abrir pull requests semanais para:

- dependências Maven do backend;
- imagens Docker do deploy Oracle;
- GitHub Actions.

Os PRs recebem labels `dependencias` e `seguranca`.

## Workflow De Segurança

Novo workflow:

```text
.github/workflows/security-analysis.yml
```

Execução:

- manual via `workflow_dispatch`;
- semanal às segundas-feiras.
- o job `Dependency-Check Maven` é opcional e só roda quando o input
  `rodar_dependency_check` for marcado manualmente.

Observação operacional: GitHub Actions só disponibiliza `workflow_dispatch` e
`schedule` para workflows presentes na branch padrão do repositório. Enquanto
esta entrega estiver apenas em `develop`, o arquivo fica versionado, mas a
execução manual/agendada do workflow de segurança só aparecerá após merge para a
branch padrão.

Jobs:

- `Dependency-Check Maven`: execução manual opcional do OWASP Dependency-Check, falhando para CVSS >= 9.0;
- `Trivy filesystem`: varre vulnerabilidades, segredos e misconfigurations com severidade alta/crítica e publica SARIF como artefato;
- `Smoke público de segurança`: valida a superfície pública atual.

Na primeira execução em `main`, o Trivy e o smoke público passaram. O
Dependency-Check foi mantido como opcional porque a atualização inicial da base
NVD excedeu 30 minutos sem `NVD_API_KEY`. Para uso recorrente desse job,
recomenda-se configurar o secret `NVD_API_KEY` no GitHub.

## Smoke Público

Script:

```bash
python3 scripts/security-smoke.py
```

Valida:

- headers defensivos em `opiagile.com`;
- `/actuator/health` público sem vazamento sensível;
- `/api/workspaces` e `/api/documents` bloqueados sem API key;
- `/api/admin/*` não público;
- `demo-rag.opiagile.com/api/workspaces` bloqueado sem API key.

O script não usa credenciais reais e não envia carga agressiva.

## Workflow Develop

O workflow `develop-ci-deploy.yml` agora também:

- valida `Caddyfile.https`;
- valida `Caddyfile.http`;
- executa `scripts/security-smoke.py` após deploy Oracle.

Isso reduz o risco de publicar uma alteração que remova headers defensivos ou exponha endpoints protegidos.

## Testes Unitários

Foram adicionados testes para garantir que:

- auditoria de API clients usa `X-Real-IP` quando informado pelo proxy;
- rate limit da demo usa `X-Real-IP` quando disponível, reduzindo risco de spoofing por `X-Forwarded-For`.

## Limitações

- Dependency-Check e Trivy dependem de bases externas e podem falhar por indisponibilidade temporária.
- Dependency-Check não roda no agendamento semanal por padrão; ele deve ser acionado manualmente, preferencialmente com `NVD_API_KEY`.
- O smoke público valida o ambiente publicado, não substitui teste autenticado com API key real.
- O workflow `Análise de segurança` só ficará disponível para execução manual/agendada após o arquivo existir na branch padrão do GitHub.
- CodeQL e upload SARIF para Code Scanning não foram adicionados nesta sprint para evitar dependência de configuração de code scanning do repositório.
