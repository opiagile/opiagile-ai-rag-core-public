# Relatório v0.7 - Core Sem Frontend

## Resumo Executivo

A v0.7 reposiciona o projeto como `opiagile-ai-rag-core`, removendo a interface gráfica do repositório e mantendo o foco em backend/API de RAG.

O repositório remoto foi renomeado para:

```text
https://github.com/opiagile/opiagile-ai-rag-core
```

## O Que Mudou

- `demo-ui/` foi removida do core.
- `landing/` foi removida do core.
- A pipeline `develop` deixou de executar builds Node/frontend.
- O README foi reescrito para refletir core API.
- A coleção Postman foi renomeada para `opiagile-ai-rag-core`.
- O `artifactId` Maven passou a ser `opiagile-ai-rag-core`.
- Defaults locais de banco/container foram renomeados para `opiagile-ai-rag-core`.
- Documentação de handoff do frontend foi criada em `docs/frontend-handoff.md`.

## O Que Permanece

- Backend Java/Spring Boot.
- Upload de documentos.
- Chunking.
- Embeddings OpenAI opcionais.
- Recuperação pgvector com fallback textual.
- Geração conversacional opcional com LLM.
- Memória conversacional.
- Triagem, lead e handoff.
- Observabilidade.
- Deploy Oracle Cloud Always Free.
- Piloto WhatsApp preservado temporariamente como referência técnica para futura extração.

## Decisões

- Frontend deve virar repositório separado, sugerido como `opiagile-rag-demo-web`.
- Gateways de canal devem virar aplicações separadas, começando por `opiagile-whatsapp-ai-gateway`.
- Este core deve priorizar qualidade de RAG, avaliação, API e operação backend.
